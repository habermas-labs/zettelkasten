## Core Concept

A Hedra is a structured interaction preset defined by the *facing* assigned to each model — the perspective, stance, or face each model turns toward the object of inquiry. The name derives from the Greek root *hedra* (face), the same root in polyhedron naming. Coined in Crosstalk Lab development as a parallel class to Tetras.

The singular is plural by nature: three models, three faces. Hedra and Hedras both valid; Hedron/Hedra explicitly rejected.

---

## Distinction from Tetras

**Tetras** define *who is at each vertex* — role and structural assignments in a four-actor model (human + 3 models). A Tetra is about position in the interaction geometry.

**Hedra** define *what face each model turns toward the problem* — perspective and stance assignments independent of vertex structure. A Hedra is about orientation to the object of inquiry.

The two classes are **orthogonal and composable**: a Tetra can run with a Hedra overlay, giving each vertex both a role and a facing simultaneously.

---

## Connection to Bakhtin

The surplus of seeing (vnenakhodimost') arises precisely from *which face is turned toward the object*. Different facings produce different surpluses — perception unavailable from any other position. Hedra operationalizes this: by explicitly assigning facings, CT creates the conditions for structured surplus generation across models.

A Hedra is not just a prompt engineering convenience. It is a deliberate epistemic geometry.

## Interaction Geometries Uniquely Enabled by CT

Hedra emerge from a broader observation: CT enables interaction geometries that are structurally impossible or prohibitively contrived for human communication. Captured here as a generative list — not exhaustive.

**The Crosstalk Move** — simultaneous bilateral exchange where two models each receive the other's response as input at the same moment. No signal loss from simultaneity (models read text, not audio). The human analog requires a conductor, isolation of both parties, simultaneous collection, and relay — so contrived it is effectively nonexistent in practice. CT does it natively. The asymmetry present in sequential directed routing is structurally absent here.

**The 3x2 Perspective Matrix** — each model is assigned two perspectives and directed to respond from each in sequence; all execute in parallel. Result: six responses from one prompt. Two axes of contrast simultaneously available: within each model (same architecture, two positions) and across models (different architectures, same position pair). Can be extended to round-robin perspective rotation — Claude argues A then B, ChatGPT argues B then C, Gemini argues C then A — creating a geometry where each model's second response is another model's first response territory.

**Simultaneous multi-perspective parallel response** — all models answer the same prompt simultaneously from their own positions, without exposure to each other's responses first. The human analog (peer review, panel discussion) is sequential and socially contaminated — participants hear each other and self-censor or conform. CT's version is structurally clean.

**Symmetric role reversal** — two models simultaneously swap positions, each arguing the other's prior stance, neither having seen the other's new response yet. Humans can do role reversal but not symmetrically and simultaneously — the second person to switch has already heard the first.

**Stateless re-entry** — models enter each exchange without ego investment in prior positions. Not just a personality difference — a different interaction geometry. The social/emotional residue that shapes human dialogue trajectory is structurally absent.

**The participant-observer** — a model holds the External Observer role while remaining a full content participant. In human interaction these roles are mutually exclusive: you are either in the conversation or watching it. CT can hold both simultaneously in the same vertex.
