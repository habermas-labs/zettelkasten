## The Claim

The horizontal input band in Crosstalk Lab — containing the prompt textarea, session vault chip, and intent controls (Parallel, Conductor, Direct To, Include) — is not a submission interface. It is the surface of user agency: the place where the conductor's intentions about the epistemic encounter become executable.

The distinction matters. A submission interface receives content. The CT input layer receives intent — including metacognitive decisions about how the dialogue will be structured, which voices will participate, what context will travel forward, and what shape the inquiry will take. The prompt itself is only one expression of that intent. The routing and mode controls are equally intentional acts, encoding the conductor's epistemic strategy before a word is typed.

---

## Reframing the Controls

What appears to be software configuration is, from the user's perspective, the exercise of epistemic judgment.

Choosing Parallel versus Conductor is not selecting a send behavior — it is deciding whether the session will generate independent perspectives or build a sequential exchange. Choosing Include is not attaching context — it is a deliberate act of synthesis, deciding what prior voice should bear on the next turn. Choosing Direct To is a curatorial judgment about which model's specific capacities are most relevant to a particular question.

These are metacognitive decisions: the user reasoning about their own reasoning process before the session proceeds. The controls are the interface to the conductor's epistemic strategy, not to CT's configuration.

This framing has a direct connection to Bakhtin's concept of the author: the conductor is not merely a participant in the dialogue but the organizing intelligence that shapes its conditions. The input layer is where that authorial function is exercised.

---

## Admission Criterion

The admission criterion for the input layer follows from this reframing: a feature belongs in this band if it is an expression of user intent — not merely if it involves user-supplied content, and not if it configures CT behavior as such.

This distinction governs future layout decisions and protects the band from accumulating controls that belong elsewhere. A toggle that adjusts CT's internal behavior lives in Settings. A control that shapes how the conductor's next move will be interpreted lives in the input layer.

The session vault chip (CT-083) belongs in the input band because uploaded documents are the conductor's contributed context — active epistemic material the conductor has chosen to bring into the session. Its presence in the band signals that the conductor's contribution to a turn is not limited to the words in the prompt box.

---

## The Threshold

The carousel sequence that precedes first use lands the user at this band deliberately. The sequence completes — *triangulation · reciprocity · the Agora · calibration · the In-Between* — and delivers the conductor to the place where their participation begins. The final slide's tone should be calibrated to that moment of arrival: not the end of an explanation but the beginning of an encounter.

The input layer is the threshold, not merely the interface. The carousel ends here because this is where the user's role begins.
