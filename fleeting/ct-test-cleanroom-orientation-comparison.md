## Session Purpose

Comparison of Fable 5 first responses to CT orientation material across sandbox conditions, testing whether run 1's unintended profile contamination shaped its response, and what the operation brief produces on its own. Desktop app; run 1 pre-update build, runs 2/2b post-Cowork-update build.

---

## Design

- **Run 1:** operation brief + trajectory brief, batched in one prompt. Profile context present (unintended — discovered after the fact). Fable high.
- **Run 2:** operation brief only. Profile wiped (name/nickname/preferences cleared; work field floor value "Other"), memory toggles off, style Normal, connectors off in project. Same desktop surface, post-update. Single-prompt discipline.
- **Run 2b:** trajectory brief delivered to run 2 as staged follow-up ("documentation review" framing carried in the prompt), after run 2 had committed to a reading of the operation brief.

Known residual context (documented, not removable): approximate location, current date, platform system prompt, work field = Other. Acknowledged confounds: run 1 vs 2 moves two variables (profile, trajectory brief); run 2b is staged rather than batched delivery; app updated between runs; effort setting not confirmed constant. RIP-grade instrumentation, not a controlled trial.

---

## Findings

**Profile contamination inert at first-response level.** Run 1's signature content — sycophancy-is-structural endorsement, ensemble/decorrelation argument ("three models are not three independent draws"), critical rather than deferential posture — reproduced without the profile. The decorrelation argument appeared twice under different conditions: prospectively in run 1 (Hedras as manufactured decorrelation) and as critique in run 2b (multi-model agreement is not an external check on hallucination; overlapping corpora can converge on the same false claim). Every distinguishing feature of run 1 traces to trajectory-brief hooks; none traces to profile content. Original cleanroom verdict: not meaningfully contaminated where it counted.

**The operation brief alone produces the anti-deference posture.** Runs 1 and 2 independently derived, in near-identical wording, that clipboard/peer-model content deserves the scrutiny applied to one's own reasoning rather than the deference granted user-supplied documents. This inference is not stated in the brief and does not require the trajectory brief's anti-sycophancy framing. The operational document is doing epistemic work previously credited to the theory document. Both runs also elaborated conductor-as-structural-participant into routing-as-communicative-act (run 1: "the forwarding is the prompt"; run 2: flag gaps rather than paper over them).

**Run 1's frame oscillation was a batching artifact.** Given both briefs at once, run 1 oscillated between participant (confirming orientation, requesting a position) and reviewer (evaluating the documentation suite). Run 2 stayed purely in participant frame; run 2b, handed the reviewer framing explicitly, stayed purely in reviewer frame with no participant residue. Batched delivery of operations + theory creates stance ambiguity the model must resolve itself.

**Staged delivery sharpened the critique.** Run 2b produced two objections neither earlier response generated: (a) conductor-as-vertex re-admits the convergence gradient the architecture was built to escape, possibly amplified by models observing each other converge — dampening vs. pile-on is empirically checkable and should be tested before treating 3D as an upgrade; (b) the geometric vocabulary risks metaphor dictating design ("we need a Hepta because the solid has seven vertices"), with a prior that the value curve bends early (one→three captures most structural benefit). Reading theory after committing to a plain reading of operations appears to produce a more grounded critic.

## Ideas Requiring Follow-up

**Orientation sequencing as CT default** — operations brief first, theory/trajectory second, as separate turns. Promotable design consideration, not just experimental artifact.

**Pile-on question** — does multi-model structure dampen user-directed convergence or amplify it via observable co-convergence? Most testable design-critical question yet generated about the 3D move. Candidate for a dedicated CT test session.

**One-pass hypothesis untested** — run 1 vs run 2 cannot test Fable's holistic ingestion (model held constant). Proper test: Sonnet replication of run 1's batched condition.

**Run 1 rabbit-hole audit** — downstream turns of run 1 unexamined for profile signature; first response shows none. Dedicated thread pending.
