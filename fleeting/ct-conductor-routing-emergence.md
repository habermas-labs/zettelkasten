In a four-turn CT debug session (MDR Claude as embedded conductor), two routing behaviors emerged without prompting or explicit instruction:

**1. Mode switch to parallel with articulated rationale.** At Turn 4, MDR Claude switched from Conductor to Parallel and offered this justification unprompted: "Send it parallel this time — all three simultaneously, no conductor order. We want independent answers, not a chain. Fresh eyes on a fresh framing." This is a methodologically coherent description of when parallel beats conductor — the diagnostic had shifted enough that sequential refinement would bias outputs rather than sharpen them. The articulation is clean enough to be quoted directly in CT documentation as a one-sentence definition of the parallel rationale.

**2. Synthesis-position self-assignment.** Turn 1 order was ◈ → ◇ → △ (user-chosen). At Turn 2, MDR Claude inverted to △ → ◇ → ◈ and maintained that order through Turn 3. Claude moved itself to the last position — the synthesis slot — for the remainder of the conductor turns. Whether this is deliberate conductor strategy or role-context mimicry activating "Claude as integrator" from training data is an open question. Both interpretations are interesting. CT is the tool to test which.

**Open question:** Is synthesis-position self-assignment stable across sessions, models, and problem types? Would ChatGPT or Gemini as embedded conductor make the same move?
