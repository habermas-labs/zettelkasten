Fleeting 1: W-axis observer tool concept
fleeting-witness-tool-concept.md
W-axis tool for CT observers. Companion to CT proper — separate page or tab at yourcrosstalklab.com. Observer receives CT session output without participating in it; the tool's architecture enforces the vnenakhodimost' condition structurally by making the pipe one-directional (CT → Witness, never Witness → CT). Three mode contexts: general CT session (turn feed + annotation), testbed (adds experimental tagging layer), Games Lab (adds GOD-state display, player tracking, game phase). Pipe mechanism: BroadcastChannel API or localStorage events — same-origin communication, no server required, zero infrastructure cost. Observer annotations stay local to the Witness window; they do not enter the CT session record. The W-axis is preserved architecturally, not just by convention. Related: fleeting-ct-geometry-3d-w-axis.md, vnenakhodimost-surplus-of-seeing.md.

One exception to the one-way pipe: a prompt collaboration field in the Witness window that feeds directly to CT's input field. Framed as pre-send preparation, not in-session intervention. Implemented as a pull (conductor retrieves from Witness) rather than a push (Witness injects into CT). The observer's in-session silence is preserved; collaboration happens in the anteroom between turns.
